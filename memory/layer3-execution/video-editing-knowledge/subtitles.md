# 字幕设计原则

## 时间同步铁律

**字幕出现**：说话人第一个字发声的同时 → 字幕出现
**字幕消失**：说话人最后一个字结束的同时 → 字幕消失（可延长 0.1-0.2s 缓冲）

技术实现：使用 Whisper 词级时间戳而非估算。每个字幕行的 start = 该行第一个词的时间戳，end = 该行最后一个词的时间戳。

## 断行规则

| 条件 | 处理 |
|------|------|
| 一行 ≤ 18 个中文字 | 单行显示 |
| 一行 > 18 个字 | 在标点处断为两行 |
| 英文/数字混排 | 按字符宽度等效计算（1 中文 ≈ 2 英文） |
| 专有名词 | 不断开（如"新天地河滨花园"不跨行） |

## 知识型内容的字幕特殊性

### 数据字幕（新增）

知识型内容大量涉及数字。数字不能只靠耳朵听，必须在画面上同步出现：

| 数据类型 | 字幕处理 | 示例 |
|----------|----------|------|
| 价格/单价 | 数字加大字号、金色或高亮色 | **108,000/㎡** |
| 对比数字 | 左右分列或上下对比，差额用不同颜色 | 新天地 10.8万 vs 仁恒 9.7万 → **差额 +1.1万** |
| 百分比 | 数字 + 趋势箭头 | 跌幅 15% ↓ |
| 排名/序列 | "第X位" + 数字强调 | 天山板块 **第1** |

**原则**：观众关闭声音也能从字幕上读到核心数据。这是知识型内容和泛口播内容的本质区别。

### 文化感字幕

知识型内容中偶尔出现的文学性表达，字幕处理要不同：

- **引用/典故**：用宋体、稍小字号、加引号或斜体区分——"房子会旧，但一个愿意为品质投票的社区，不容易老"
- **地理意象**："苏州河拐进天山板块这一段"——关键词保留在字幕上比语音多停0.3-0.5秒，让观众有时间"看到"画面

### 金句字幕

金句是整条视频的"文眼"，字幕处理要让它被记住：

- 全屏居中，宋体 Bold，字号 90-110px
- 出现前留白 0.3-0.5 秒（无字幕、无数据叠加），制造注意力集中
- 金句说完后停留 1-1.5 秒再切走
- 可加极简的淡入动画（Alpha 0→100% over 0.5s）

## 安全区

```
┌─────────────────────┐
│  标题安全区 (20%)     │  ← 章节标题、系列标识
│                     │
│  内容安全区 (60%)     │  ← 数据卡片、对比图表、金句
│                     │
│  字幕安全区 (20%)     │  ← 常规字幕、数据强调
└─────────────────────┘
```

字幕距底部至少 60px（约占屏高 3%），确保在小屏手机上不被遮挡。

## 样式默认值

| 参数 | 知识型默认 | 说明 |
|------|-----------|------|
| 字体 | 思源黑体 Regular | 常规字幕 |
| 标题字体 | 思源宋体 Bold | 章节标题、金句 |
| 字号 | 60px | 比常规大2px，给阅读留余地 |
| 颜色 | 白色 #FFFFFF | 最高对比度 |
| 描边 | 黑色 2px | 浅色背景可读 |
| 阴影 | 1px | 增强立体感 |
| 位置 | 底部居中 (alignment=2) | 行业标准 |
| 数据颜色 | 金色 #FFD700 或暖橙 #FF8C00 | 关键数字专用 |

## 特殊场景

- **数字强调**：数字单独一行、加大字号、金色显示。这是知识型内容最重要的字幕技巧
- **金句**：全屏居中、宋体 Bold、最大字号、淡入动画、说完停留
- **对比数据**：左右分列或上下对比，差额用不同颜色标注
- **章节标记**：知识型内容段落清晰，每个段落开头可以有一个 2-3 秒的章节标记（如"优点""缺点""适合谁"）
- **中英混杂**：英文用半角空格隔开，保持可读性
