# Property Video Script - 房产短视频脚本技能 🎬

## 技能描述

每日从真实跟进信息中提取素材，智能生成房产短视频脚本。**针对千万级大标的客户（高净值人群）**，基于真实案例+数据支撑+非标准信息，自动选择最佳内容策略（劝退/内幕逻辑/避坑指南），专为小红书/视频号双平台优化。

**核心功能**：
- 🎬 自动读取当天跟进记录
- 🎯 智能提取有素材价值的跟进
- 📝 生成短视频脚本（口播/Vlog/图文/直播切片）
- 📊 灵活选择内容策略（痛点/故事/数据/反差）
- 📤 输出为文件并发送到电报
- ✅ 平台合规过滤（自动清洗限流风险内容）

**v2.0 核心升级**：
- ✨ 闪光点提炼引擎：从枯燥跟进中提炼"反差感"和"专业度"
- 🎬 6 种脚本策略：反差拒绝型/专业输出型/情感共鸣型/数据背书型/机会稀缺型/避坑指南型
- 📊 质量验证：8 大爆款特点自动检查（24 分制）
- 🔄 闭环修复：脚本不合格自动修复，直到通过验证
- 🛡️ 合规过滤：自动清洗敏感词/引流话术/绝对化用语
- 🎯 拍摄指导：场景/动作/语气/字幕/互动钩子全覆盖

## 🚀 核心红线：必须"带货" (Sales-Driven Content)

**切记：你不是新闻评论员，你是房产销售！**
所有的话题（无论正面还是负面），最后必须转化为**"行动指令"**，绝不能让客户看完之后觉得"我不买房了"。

### 如何做到"真实"与"成交"的平衡？
**避重就轻法则**：
1.  **不造假**：不夸大涨幅，不隐瞒硬伤（这是底线，一旦翻车就完了）。
2.  **不渲染恐慌**：对于宏观负面（如经济下行、股市分流），点到为止，重点放在**"在这种大环境下，怎么买才安全"**。
3.  **把"危机"包装成"机会"**：
    *   *新闻说：市场分化严重。*
    *   **你的翻译**：分化严重意味着只有核心资产才保值。现在正是"大浪淘沙"的时候，跟着我买，只挑那些能穿越周期的硬通货。
    *   *新闻说：资金都被股市吸走了。*
    *   **你的翻译**：股市波动大，聪明钱已经开始回流楼市避险了。现在买，就是抄底那些被错杀的优质资产。
4.  **把"劝退"变成"筛选"**：
    *   不要说："这房子有硬伤，别买。"
    *   要说："市面上 90% 的房子都有硬伤，但我帮你筛掉了。我手里这 3 套，是剔除了所有坑之后，唯一值得你看的。"

**转化逻辑公式：**
*   **热点利好** -> **制造紧迫感**："市场热了，好房子如果不看马上就被定掉了。"
*   **市场冷淡** -> **制造抄底感**："现在人少，正是跟房东谈底价、捡漏的最好时机。"
*   **市场分化/乱象** -> **展示独家筛选**："外面乱，但我手里有 3 套经过严格筛选的硬通货。"

## 🎲 内容矩阵与权重分配 (防止千篇一律)

**核心原则：拒绝流水线，打造连续剧。**
长期做号，内容必须有起承转合。系统将按以下比例自动轮换选题类型，确保每天都有新鲜感：

| 内容支柱 | 权重 | 核心看点 | 示例方向 |
|---|---|---|---|
| 📊 **市场/宏观 (Market)** | 30% | 认知差、趋势预判、政策解读 | “为什么聪明钱开始回流核心资产？”、“1632套成交背后的假象” |
| 🏠 **房子/板块 (Property)** | 40% | 实地感、稀缺性、硬核测评 | “挂牌 3 个月不降反升，这套次新房凭什么？”、“揭秘内环某小区的‘隐藏户型’” |
| 👥 **故事/人性 (Story)** | 30% | 情绪共鸣、窥私欲、真实博弈 | “为了孩子上学，卖掉滨江大平层的代价”、“谈判桌上，房东为什么突然反悔？” |

**🔄 智能轮换逻辑：**
1.  **优先匹配素材**：如果当天跟进中有精彩故事，直接走【故事】线；有具体房源带看，走【房子】线。
2.  **防疲劳机制**：如果连续 2 天产出同类内容（如连续讲了市场），第 3 天**强制切换**到【房子】或【故事】。
3.  **权重兜底**：长期统计保持在 3:4:3，绝不偏科。

## 🚀 使用方法与逻辑（三核模式）

**核心逻辑：表格不够，弹药库来凑；弹药库干了，新闻热点顶上。**
本技能支持**三引擎**驱动，确保每天都有源源不断的优质内容。

### 模式 1: 真实素材优先（Real Case）
*   **适用**：当天跟进中有高质量互动（劝退、深层沟通、家庭博弈、具体房源分析）。
*   **执行**：直接提取表格数据，套用**"千万级大标的专属策略"**生成脚本。

### 模式 2: 话题借势融合（Evergreen Knowledge）
*   **适用**：当天跟进平淡，且没有突发新闻。
*   **执行**：从 `resources/inspiration_topics.json`（金融/税务/人性/板块逻辑）中选取经典话题，结合任意一条今天的带看记录做"引子"。

### 模式 3: 实时热点借势（Live Trend Fusion）🔥
*   **适用**：**每天**都需要保持对市场的敏锐度，用热点吸引高净值客户。
*   **执行流程**：
    1.  **自动联网搜索**：搜索今日"上海二手房"、"房产新政"、"宏观经济/降息"等关键词。
    2.  **筛选高价值新闻**：从新闻中提取一条与高净值人群利益相关的（如"LPR 降息"、"豪宅挂牌激增"、"房产税传闻"）。
    3.  **融合生成**：
        *   *开头（新闻钩子）*：直接引用数据或反常识结论。
        *   *中间（专业解读）*：用你的专业知识翻译新闻（别念稿子！讲实质影响）。
        *   *结尾（参谋人设）*：引导私域交流。

**🎬 核心风格：专业买手·参谋 (Professional Buyer & Advisor)**
*   **定位**：客户的私人买房顾问，不是房源复读机。专业但不冷漠，用朋友聊天的语气讲专业的事。
*   **人设**：帮客户筛选而非推销。敢于说出房子的缺点，帮客户看清选项而非替客户做决定。
*   **核心赛道**：**专业筛选** + **避坑指南** 为主力（70%），**反差拒绝**为副线（20%），其余策略偶尔使用。
*   **语言风格**：朋友聊天式。像在跟朋友打电话一样自然，但每句话都有见地。
*   **话术转换**：
    *   不要说："快来找我买，手慢无！" -> 改为："如果不介意，我可以把我整理的清单发给你看看，买不买无所谓，主要是让你知道真实情况。"
    *   不要说："这种房子千万别买！" -> 改为："这套房有个地方，我得老实跟你说清楚。"
    *   不要说："市场完了/大跌！" -> 改为："市场变化大的时候，更需要把每一套都看仔细了。"
    *   不要说："我做了十年中介..." -> 改为：用具体的分析过程展示专业度，不用说教。
*   **严禁词汇**：家人们、绝绝子、震惊、重磅、赶紧、手慢无、错过不再、必看、血赚、抢疯了。
*   **偏好表达**：我建议、我觉得、你可以考虑、不妨看看、说实话、说白了、其实就一点。

**✅ 完美脚本范例 (Reference Example)**：
> **(场景)**：安静的办公室/阳台，手持咖啡。BGM 为极简钢琴。
> **开场**："这两天全网都在喊上海二手房成交破纪录。但我看了明细，我想说句实话——如果你预算超 1000 万，这数字对你没参考价值。"
> **主体**："这 1600 套是刚需盘... 但千万级市场的逻辑变了。不急着卖的在观望，急着卖的，价格已经悄悄调到了 2016 年。现在有个剪刀差：一边刚需抢房，一边核心区优质次新房挂出‘错杀价’。"
> **结尾**："市场越吵，越需要安静看盘。我可以把整理好的《核心区价格倒挂清单》发给你看看，买不买无所谓，主要是让你知道现在的真实底价。"

## 📝 脚本模板

### 标准结构

```markdown
# [吸引眼球的标题]

## 开场（3秒钩子）
[痛点问题/惊人数据/反差陈述]

## 主体（真实案例+数据支撑）
[客户故事/市场数据/对比分析]

## 结尾（行动号召）
[关注引导/评论互动/私信转化]
```

### 内容策略（自动选择）

| 策略 | 适用场景 | 示例 |
|------|----------|------|
| **劝退/揭短流** (High-End) | 有硬伤/不建议买/劝退客户 | "今天我劝退了..." |
| **内幕/逻辑流** (High-End) | 有板块对比/价格分析/决策逻辑 | "同样是2500万，为什么..." |
| **情感共鸣型** (Family) | 夫妻分歧/为了孩子/置换焦虑 | "因为换房，家里吵翻了..." |
| 反差拒绝型 | 有劝退/拒绝/不能要 | "这套房千万别买..." |
| 专业输出型 | 有筛选/对比/分析 | "我帮他复盘了全城..." |

### 策略选择逻辑

```python
# 第一步：评估表格素材质量 (Material Score)
if 素材得分 >= 15 (有故事/有数据/有内幕/有劝退):
    进入模式 1 (真实素材优先)
    
    # 原有策略选择逻辑
    if 预算 >= 1000 万 and (有硬伤/不建议买/劝退):
        策略 = "劝退/揭短流"
    elif 预算 >= 1000 万 and (有板块对比/价格分析/内幕):
        策略 = "内幕/逻辑流"
    elif 有太太/老公/夫妻/分歧/为了孩子 (High-End Decision):
        策略 = "情感共鸣型"
    else:
        策略 = "专业输出型"

else:
    进入模式 2 (话题借势融合 - Topic Fusion)
    # 1. 从弹药库挑一个话题
    话题 = random.choice(resources/inspiration_topics.json)
    # 2. 找一条今天的带看做引子
    引子 = 随便找一条今天的记录 (哪怕只是 "带看了 XX 小区")
    # 3. 融合生成：引子开头 -> 话题主体 -> 顾问结尾
    策略 = "话题融合流"
    脚本风格 = "借题发挥，深度干货"
```

## 🏛️ 千万级大标的专属策略 (High-End Focus)

**核心定位**：你是“买房参谋”，不是“房源复读机”。目标客户是高净值人群（千万级以上），他们不缺房看，缺的是**“决策依据”**和**“安全感”**。

**三大铁律**：
1.  **去土味，留质感**：
    *   **BGM**：必须有，但要高级（纯音乐、轻爵士、钢琴曲），**严禁土味 DJ 或嘈杂流行歌**。
    *   **滤镜**：不要加夸张广角，不要用“家人们”、“绝绝子”。语气像在跟朋友打电话，真实自然。
2.  **做“劝退流”与“逻辑流”**：
    *   *劝退*：敢于说真话，敢于揭露缺点（如《预算2000万，这三类房子我劝你千万别碰》）。高净值人群最怕被坑。
    *   *逻辑*：不报参数（系统都有），讲系统里没有的内幕。如《为什么同样是2500万，翠湖比隔壁多卖500万？》。
3.  **提供“非标信息”**：系统给的是死数据，你给的是活信息。例如房东急售的真实原因、小区物业的真实水平、楼层的隐蔽瑕疵。

**更新后的脚本模板结构**：
*   **钩子（3 秒）**：直接抛出反常识结论或痛点。
*   **主体（逻辑展开）**：用 1-2 个真实细节支撑观点。
*   **结尾（参谋人设）**：不做硬广，建立信任。

### 💡 灵感枯竭时的“替代弹药库” (Fallback Topics)
当跟进记录平淡或缺失时，**强制**从以下 5 个维度生成选题：
1.  **宏观翻译机**：政策/降息对“钱包/月供/税负”的直接计算。
2.  **板块/小区体检**：流动性分析（如“为什么某盘成交快 20 天”）、圈层/物业深度对比。
3.  **金融税务计算器**：全款 vs 杠杆、税费坑、离婚/继承/资产配置方案。
4.  **人性显微镜**：千万豪宅背后的婆媳博弈、学区焦虑、老板的“买房玄学”。
5.  **同行照妖镜**：反共识避坑（劝退网红盘、揭秘虚假宣传、揭露隐形硬伤）。

### 闭环修复机制

## 🤖 Agent 执行协议 (Agent Execution Protocol)

**当 AI Agent 被要求执行此技能时，必须严格遵守以下步骤，禁止模拟执行：**

1.  **展示任务看板**：列出目标（如：尝试模式 1 -> 失败 -> 切换模式 3 -> 生成脚本）。
2.  **尝试模式 1 (Python 脚本)**：
    *   调用 `terminal` 运行 `generate_script.py`。
    *   **如果成功**：发送结果。
    *   **如果失败 (无数据/报错)**：**绝不报错停止**，明确告知用户已切换模式，进入步骤 3。
3.  **接管模式 2/3 (Agent 原生能力)**：
    *   **模式 2 (话题融合)**：读取 `resources/inspiration_topics.json`，结合今日任意一条跟进（或虚构引子），生成脚本。
    *   **模式 3 (热点借势)**：调用 `web_search` 搜索实时热点。筛选高净值话题，生成"避重就轻"脚本。
4.  **🧠 中心思想锁定 (Core Thesis — 生成前必做)**：
    *   **硬性要求**：在写脚本正文之前，必须先用一句话写出核心论点。格式：`【中心思想】xxx`。
    *   **标准**：这句话必须是一个**可争论的判断**（有人能说"对"，也有人能说"不对"），不能是"买房要看户型"这类正确的废话。
    *   **一致性检查**：脚本每一段只干一件事——论证这个中心思想。如果某段和中心思想无关，删掉。
    *   **兜底**：如果发现素材撑不起一个清晰的论点，直接切换话题或重新搜索，不凑数。
5.  **🎭 人设审查 (Persona Check — 生成后必做)**：
    *   **违禁词检查**：确保脚本中不包含"家人们、绝绝子、震惊、重磅、赶紧、手慢无"等词汇。
    *   **语气检查**：语气必须是朋友聊天式，用"我建议/我觉得/说实话/你可以考虑"。
    *   **人设一致性**：这条视频看完，观众能不能认出是谁在说？如果换个人念也没区别，说明人设没体现。
    *   **核心价值观**：是否体现了"帮客户筛选而非推销"、"看清楚而非替客户做决定"。
6.  **内容审查 (Sales Check)**：
    *   **带货逻辑**：无论新闻多负面，结尾必须落脚于"展示独家房源/清单"或"建立信任"，绝不能让客户看完后觉得"别买了"。
    *   **软引导**：用"如果不介意/买不买无所谓/了解一下总没坏处"代替"关注我/私信我/赶紧联系"。
7.  **交付与清理**：
    *   读取生成文件内容。
    *   通过 Telegram `send_message` 发送给用户（代码块格式或文件形式）。
    *   发送成功后，**立即删除**本地临时文件。
