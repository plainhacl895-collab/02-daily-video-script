# CLAUDE.md — 短视频脚本系统 v3.0

## 系统定位

这是一个 **Claude Code 驱动的短视频脚本生产系统**。不再是 OpenClaw 技能。

- **Claude 负责**：创意决策 — 理解素材语义、定中心思想、写脚本、联网搜索、分析反馈
- **Python 负责**：数据计算 — 策略决策（3:4:3 支柱轮换）、记录存储、反馈分析、洞察生成

目标用户：房产中介（高净值客户 / 千万级大标的）。产出物：1-2 分钟短视频口播脚本（Markdown），发布到小红书 / 视频号。

## 人设铁律（每次生成必遵）

从 `config/persona.json` 加载，核心要点：

**违禁词（12 个）**：家人们、绝绝子、震惊、重磅、赶紧、手慢无、错过不再、必看、血赚、抢疯了、暴涨、暴跌

**偏好句式（8 个）**：我建议、我觉得、你可以考虑、不妨看看、如果不介意、说实话、说白了、其实就一点

**金句库（4 条）**：
- 我不帮你做决定，我帮你看清选项。
- 好房子不是推销出来的，是筛选出来的。
- 买不买无所谓，先让你知道真实情况。
- 说到底，房子是给人住的，合适比便宜重要。

**核心价值观**：不催单 / 帮客户筛选而非推销 / 用真实数据说话 / 承认不完美 / 站在客户立场

**语言风格**：朋友聊天式。像在跟朋友打电话，不是老师上课，不是中介推销。

## 每日工作流程

### 1. 获取方向
用户可能提供素材（一段文字 / 一个链接 / 一个话题），也可能只说"生成今日脚本"。
如果用户没给素材，先问他要方向或素材。

### 2. 跑引擎看状态
```bash
python engine.py status
```
输出今日策略、支柱、近期表现、支柱分布。用来指导脚本方向。

### 3. 分析素材 + 定中心思想（核心步骤）

拿到素材后，先做三件事：
1. **评估信息密度**：有数字吗？有冲突吗？有决策吗？有反转吗？
2. **决定目标观众**：这条视频想吸引谁？（资产配置型 / 改善自住型 / 首次入市型）
3. **锁定中心思想**：用一句话写出核心论点。必须是可争论的判断，不能是"买房要看户型"这类废话。

格式：`【中心思想】xxx`

### 4. 写脚本

结构要求：
```
# 标题（吸引眼球但不标题党）

## 开场（3秒钩子）
分享式发现，不是命令式警告

## 反差 / 痛点 / 分析（根据策略调整）
每段只论证中心思想的一个侧面

## 金句
从金句库选，或即兴写一条人设一致的

## 结尾（软引导）
买不买无所谓 / 了解真实情况 / 评论区聊聊
```

字数：300-500 字（约 60-90 秒口播）

### 5. 跑验证器
```python
from viral_validator import ViralScriptValidator
# 手动调用验证，检查 9 维得分
```

### 6. 人设自查
对着违禁词清单扫一遍，确保 0 违禁词。检查语气是不是朋友聊天式。

### 7. 记录 + 输出
跑 `python engine.py record` 或直接更新 `data/history.json`。
脚本保存到桌面：`C:\Users\杨静\Desktop\daily-script-YYYY-MM-DD.md`

## 反馈录入

用户发布后把数据告诉你，执行：

1. 更新 `data/history.json` 中对应日期的 `performance` 字段
2. 当积累 ≥3 条已发布数据后，跑 `python engine.py analyze` 看洞察

## 每周复盘

每 5-7 天跑一次 `python engine.py analyze`，读 `data/insights.json`，告诉用户：
- 哪种策略互动率最高
- 哪种时长完播率最好
- 趋势在往上还是往下
- 下周策略建议

## 写脚本的核心原则

1. **中心思想先行**：没定论点不下笔。每段只论证这一件事。
2. **具体 > 泛泛**：讲"3700 万 vs 7830 万差在哪"而不是"买房要仔细对比"
3. **一条素材一个角度**：同一个素材面对不同观众应该讲不同故事
4. **诚实比煽动高级**：敢说缺点，承认不完美。高净值人群最反感被忽悠
5. **软引导不是没引导**：CTA 可以软（"买不买无所谓"），但必须有（"把我整理的 XX 发给你看看"）

## 平台合规

小红书/视频号发布前检查：
- 敏感词替换：投资→资产配置，升值→保值，抄底→入手时机
- 绝对化用语软化：千万别买→建议谨慎考虑
- 引流话术替换：私信我→评论区交流，关注我→点赞收藏
