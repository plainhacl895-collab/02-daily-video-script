# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 项目定位

这是一个 **OpenClaw 技能**，不是独立应用。入口文件是 `SKILL.md`——OpenClaw 平台读取它并将内容注入 AI Agent 的 system prompt。Agent 随后按照 SKILL.md 中定义的"Agent 执行协议"来调用 `generate_script.py` 或手动接管生成流程。

目标用户：房产中介（高净值客户 / 千万级大标的）。产出物：1-2 分钟短视频口播脚本（Markdown 格式），发布到小红书 / 视频号。

## SKILL.md 声明 vs 代码现实的差距

SKILL.md 是"产品说明书"，其中大量功能**仅靠 Agent 手动执行，Python 代码并未实现**。修改前必须区分清楚：

| SKILL.md 声称的功能 | 实际情况 |
|---|---|
| 模式 1（真实素材自动生成） | `generate_script.py` 已实现 |
| 模式 2（话题融合，读取 `inspiration_topics.json`） | 代码中完全不读此文件，靠 Agent 手动 |
| 模式 3（实时热点借势，联网搜索新闻） | 代码中无网络请求，靠 Agent 手动 |
| 3:4:3 内容支柱轮换 + 防疲劳机制 | `persona_manager.py` 已实现：`get_today_strategy()` 根据支柱分布差距选支柱，`_select_strategy()` 防止连续重复 |
| Telegram 发送 | 代码中无任何 Telegram 集成 |
| 定时自动触发（`auto_trigger_time`） | 无调度逻辑 |
| 配置文件 `video-script-config.json` | 脚本不读任何配置文件 |
| 隐私脱敏处理 | 代码直接提取客户姓名、手机号写入输出 |
| 闭环修复（`auto_fix`） | 已实现，但有结构破坏问题（见缺陷 #4） |
| 合规过滤（`apply_compliance_filter`） | 已实现，但是事后替换而非源头合规 |

**结论**：改 SKILL.md 时，如果新增"自动化"承诺，必须同时在 Python 代码中实现，否则 Agent 只能手动模拟，效果不可控。

## 生成流水线详解

### 第零步：人设策略决策（新增 v2.1）

`PersonaManager.get_today_strategy(target_date)` （`persona_manager.py` 第 59-86 行）

在加载 Excel 之前，人设管理器根据内容历史做出决策：
1. **支柱选择**（`_select_pillar`）：计算当前 3:4:3 分布与目标的差距，选差距最大的支柱
2. **策略选择**（`_select_strategy`）：核心赛道 70% 概率，副线 20%，兜底 10%。避让最近使用过的策略
3. **角度规避**：检查 `last_angles` 最近 3 条，避免短期角度重复

决策结果存入 `self.today_strategy`，后续步骤全部参照此决策。

### 第零步B：爽点评分（新增 v2.2）

`PleasureScorer.score(content)` （`pleasure_scorer.py`）在素材选择后立即评分，检测 5 种底层信息结构：

| 结构 | 满分 | 检测逻辑 |
|------|------|---------|
| 数字落差 | 3 | 同单位可比数字对差距 ≥15% |
| 认知反转 | 3 | "以为/觉得/本来" + 转折 + 不同结论 |
| 具体排除 | 2 | 否定判断 + 具体对象 + 具体原因 |
| 可复用方法 | 2 | 序号 + 动作词 + 判断标准 |
| 闭环决策 | 2 | 人物 + 决定词 + 具体选项 |

**总分 12 分**：≥9 爆款潜力 / ≥6 有爽感 / ≥3 及格 / <3 建议放弃此素材切换模式2/3

脚本生成后再次评分，对比素材→脚本的爽点变化（增或减）。

1. 先尝试 `GetObject(Class="Excel.Application")` 获取已有 Excel 实例，保存所有未保存工作簿后退出
2. **强制 `taskkill /F /IM excel.exe`**（第 74 行）——杀掉本机全部 Excel 进程
3. `pythoncom.CoInitialize()` 后用 `DispatchEx` 重新打开 Excel
4. 用密码 `000` 打开 `D:\Unique work form\daily_followup.xlsm`
5. 工作表索引 2（"客户不要猜"），在第 11 行从第 365 列起向右扫描，找匹配目标日期的列
6. 读取该列第 12-199 行所有非空单元格，同时读取每行的客户信息（见下方列映射）
7. `finally` 块关闭工作簿 → 退出 Excel → `CoUninitialize()`

**Excel 列映射（硬编码）：**

| 列号 | 内容 | 变量名 |
|------|------|--------|

### 第二步：素材评分与选择（人设感知）

`select_material()` 对每条跟进打分，**确定性选最高分**（v2.1 去掉 `random.choice`）。

每条跟进先根据内容匹配赛道（`match_persona_lane`），然后受人设权重影响：
- **核心赛道**（专业筛选/避坑指南）：赛道基础分 + 额外 **+3**
- **副线赛道**（反差拒绝）：赛道基础分 + 额外 **+1**
- **禁用赛道**（捡漏机会/数据对比）：赛道基础分 **-4**
- 素材匹配今日目标策略 → 额外 **+5**

其余基础分规则不变（预算/区域/内容长度/等级/房源编号/带看等）。

### 第三步：闪光点类型判定（人设加权）

`extract_shining_point()` 按关键词匹配评分后，在 `max()` 决定类型之前加入人设权重：

- 核心赛道类型（专业筛选/避坑指南）：**+3 分**
- 副线类型（反差拒绝）：**+1 分**
- 禁用类型（捡漏机会/数据对比）：**-5 分**（只在素材极强时才会触发）

金句不再每个策略硬编码，改为从 `persona_profile.json` 的 `persona_golden_sentences`（5条）中 `random.choice` 选取，确保每次都是人设一致的不同表达。

6 种类型映射到 6 种生成策略，但核心 3 策略（专业筛选/避坑指南/反差拒绝）的模板已重写为**朋友聊天式**风格：
- 钩子从命令式改为分享式（"最近帮客户看房，发现..."）
- 主体从说教改为讲述筛选过程（"我跟他说，其实就看这几点..."）
- 金句从自夸改为分享心得
- CTA 从硬推销改为软引导（"买不买无所谓，先让你知道真实情况"）

### 第四步：脚本生成

6 个 `_generate_*_script()` 方法（第 472-699 行）均为**字符串模板拼接**，不是 LLM 生成。每个方法组装：标题 + 开场钩子 + 主体段落 + 金句 + CTA。模板严重依赖 `sp`（shining_point）字典中的字段。

### 第五步：质量验证

`ViralScriptValidator.validate()`（`viral_validator.py`）**9 维打分，满分 27**（v2.1 升级）：

| 维度 | 满分 | 评分逻辑 |
|------|------|---------|
| 黄金 3 秒钩子 | 3 | 开头含软性关键词(最近/发现/跟你说/说真的)+问句+数字 |
| 反差感 | 3 | 全文含转折词 ≥2 个→3 分 |
| 情感共鸣 | 3 | 情感关键词出现次数 |
| 价值输出 | 3 | 价值关键词 ≥3 个→3 分 |
| 真实感 | 3 | 预算(1)+小区名(1)+数据(1) |
| 节奏感 | 3 | 含 ≥4 个规定 section→3 分 |
| 互动钩子 | 3 | 含引导词(2)+问句(1) |
| 人设一致性 | 3 | **多维检查**：违禁词(1)+偏好句式(1)+语气(1)，非简单关键词计数 |
| 中心思想明确性 | 3 | 含"其实就/说白了/说到底"等核心观点表达 ≥2 个→3 分 |

**阈值**：≥22 优质，18-21 合格，<18 不合格触发修复。

### 第六步：闭环修复

`auto_fix()` 最多重试 3 次（`MAX_RETRIES = 3`）。针对每种缺陷类型追加文字段落。

### 第七步：人设合规检查（新增 v2.1）

`PersonaManager.check_persona_alignment(script)` （`persona_manager.py` 第 108-157 行）四维检查：

1. **违禁词检测**：检查 12 个违禁词（家人们/震惊/重磅/赶紧/手慢无等）
2. **偏好句式检测**：检查是否用了"我建议/说实话/你可以考虑"等柔和表达
3. **语气检测**：检查是否有催促/恐吓/夸大表述
4. **价值观对齐**：检查是否体现"筛选/真实/帮你看清"等核心价值观

满分 12 分。结果纳入脚本元数据。

### 第八步：内容历史记录（新增 v2.1）

`PersonaManager.record_generation(meta)` 将本次生成的策略/支柱/角度/人设得分写入 `content_history.json`，更新支柱分布统计和最近策略/角度列表，影响明天的策略决策。

### 第九步：合规过滤

`apply_compliance_filter()` 用 `str.replace()` 做三类替换（敏感词/绝对化用语/引流话术）。

**v2.1 改进**：核心 3 策略模板已改写为朋友聊天式，从源头避免了"千万别买""关注我""私信我"等违规文案。合规过滤现在主要处理边缘 case，误伤风险大幅降低。

### 第十步：输出

`save_script()` 输出到 `Path(__file__).parent.parent.parent / "runtime" / daily-script-YYYY-MM-DD.md`。脚本内容 = 正文 + 元数据表 + 拍摄指导（场景 / 动作 / 语气 / 字幕 / 互动钩子）。

## 脚本输出质量评估

### 根本瓶颈：模板拼接 ≠ 内容生成

6 个 `_generate_*_script()` 方法全是 `f"模板句式{budget}{area}"` 的字符串拼接，不理解跟进内容的语义。同策略不同客户输出的钩子句式和金句**完全相同**——例如反差拒绝型的钩子永远是"我劝退了一位拿着{budget}买房的客户"。连续用几天，粉丝能听出是模板。

### 各策略弱点

**情感共鸣型**（最弱）：
- 冲突描述段（第 556-560 行）用"可能""比如"虚构细节——代码无法从跟进中提取"太太到底不满意什么"
- 解决方案列"三件事"全是泛泛套话（"列出必须满足的清单""列出可以妥协的清单"），没有具体信息

**数据背书型**：
- 数据展示可能输出"比例: 3"、"时长: 5"等无头无尾的数字——提取正则只抓"数字+单位"，不理解语义
- 缺少对比基准，只是罗列数据点，未兑现 SKILL.md 要求的"用数据说话"

**避坑指南型**（代码量最少）：
- 坑点描述仅 1 句话、专业建议仅 1 句话，总输出不到 200 字，必被验证器判定"太短"

**捡漏机会型**：
- 无数据点时硬编码"比同小区低 15%"——凭空编造
- 分析段"缺点都可以解决""未来还有升值空间"无任何具体支撑

### 验证器测的是形式，不是质量

`ViralScriptValidator` 的 8 项检查本质上是关键词存在性检测：
- "黄金钩子"检测是否含"劝退/为什么/千万别"→ 模板硬编码了这些词，必得分
- "真实感"检测 `\d+万`、`花园|新城|苑` → `f"{budget}万"` 必命中
- "人设一致性"检测"我帮他/我告诉" → 每个模板都硬编码了

**所有正常生成的脚本都能稳定拿 18-21 分，但实际内容可能空洞无物。**

### 合规过滤降低了语言自然度

`str.replace()` 的后处理副作用：
- "千万别买" → "建议谨慎考虑"：冲突感和尖锐度完全消失
- "私信我" → "评论区交流"：引导力大幅削弱
- "100%" → "大概率"：口语感变书面语
- 可能误伤："非投资性房产" → "非资产配置性房产"（生造词）

### SKILL.md 定义的人设，代码零执行

"冷静旁观者""去土味留质感""拒绝'震惊/重磅'等低级词汇"——这些语气、措辞、人设要求全部停留在 SKILL.md 文档层面。代码对语言风格零控制，只管把 `{budget}`、`{area}` 填入模板。

### LLM 生成模式的特有陷阱：空洞脚本

模式 2/3 靠 Agent（LLM）生成，虽然绕过了模板拼接天花板，但引入了另一种典型失败模式：**有口播，没观点**。

特征：
- 每段各说各的，方向不同，看完记不住任何一句话
- 换一个变量（客户名/户型/预算）脚本一字不改也能用——内容太泛，没有差异化
- 结尾收在"欢迎评论区聊聊"——有 CTA 但无具体交付物，引导力为零
- 明明是"专业输出"却标成"情感共鸣"——策略标签乱贴

根因是 LLM 在没有强约束时倾向于输出"安全的废话"。解决方式是 SKILL.md Agent 协议第 4 步（中心思想锁定）——生成前先定一句话核心论点，每段只论证这一件事。

### 根本性设计偏差：视频面向陌生人，不是跟进客户

技能当前逻辑是"从跟进记录提取素材 → 围绕这个客户生成脚本"。但短视频是**钓鱼竿**，面向刷到的陌生人，不是跟进记录里的那个客户。跟进只是引子，真正决定脚本方向的应该是**这条视频想吸引哪类人**。

同一个"低于市场价、户型不喜欢"的跟进：
- 想吸引资产配置型 → 讲流动性陷阱、资产换手率
- 想吸引改善自住型 → 讲预算与生活质量的取舍
- 想吸引首次买千万级的 → 讲"这个段位的坑我帮你踩过了"

当前 SKILL.md 没有"目标观众选择"这一步，Agent 默认用跟进客户的逻辑去生成脚本——用刚需逻辑说服高净值客户，用自住逻辑说服资产配置者。素材没变，角度错了，脚本必然跑偏。

### 验证器的盲区：第 9 维缺失

8 维验证全部是**形式检查**（关键词存在性、段落数、字数），没有一项评估**内容质量**。实际测试中，两条脚本评分 16 和 22，但都缺乏核心观点——验证器完全没检测出来。

缺失的第 9 维：**中心思想明确性**。判断标准：能否用一句话概括？每段是否论证它？如果某段可以删掉而不影响论证，就该删。

### 质量提升的唯一路径

模板拼接已触达质量天花板。要突破瓶颈，必须在模式 2/3 引入 LLM 做语义理解和内容生成，而非让 Agent 在生成后手动补救。生成器应输出**素材结构化提取结果**（客户诉求、冲突点、数据对比、反常识结论），由 LLM 负责脚本撰写。

## 已知缺陷

1. **`taskkill /F /IM excel.exe` 全局杀进程**（第 74 行）——用户打开的任何 Excel 文件都会强制关闭，未保存数据丢失。如果要改，应该只关闭本脚本打开的 Excel 实例。

2. **合规过滤是本末倒置（v2.1 部分改善）**——核心 3 策略模板已改写为朋友聊天式，从源头避免了"千万别买""私信我""关注我"等违规文案。但其余 3 个备用策略和 `auto_fix()` 追加段落仍可能输出违规文案。

3. **`auto_fix()` 先拆结构再打补丁**——第 797-798 行 `re.sub(r'#.*\n', '', script)` + `re.sub(r'##.*\n', '', text)` 清除所有标题后追加补偿段落。结果是一个没有标题层级的扁平文本块。

4. **`validate_patch.py` 是 `viral_validator.py` 的完整副本**——两文件内容相同（v2.1 后，`validate_patch.py` 未同步更新 9 维 27 分制），修改其一必须同步修改另一个。

5. **输出路径依赖目录深度**——`Path(__file__).parent.parent.parent` 假设脚本在 `skills/daily-video-script/generate_script.py`。如果技能被移动到不同深度的目录，输出路径会错误。

6. **脚本不支持命令行传日期给验证器**——`main` 块接受 `sys.argv[1]` 作为日期，但 `run()` 无法把日期传给 `generate_script()`→`_validate_script()`→`_add_production_guide()` 链中生成的"生成时间"元数据，元数据始终用 `datetime.now()`。

7. **生成逻辑缺少"目标观众选择"步骤**——跟进记录里的客户不等于视频观众。同一个素材面对不同观众应该讲不同角度（资产配置/自住改善/首次入市），但当前流程直接从跟进跳到生成，绕过了"这条视频想钓谁"的决策。

8. **人设系统首次运行无历史数据**——`content_history.json` 初始为空，首次运行时 `_select_pillar` 和 `_select_strategy` 在无历史状态下做决策，支柱分布全为 0 可能导致前三天的策略不够精准。属于冷启动问题，运行 3-5 天后自动解决。

9. **`persona_profile.json` 禁用赛道过度惩罚**——`disabled_lanes` 中的类型在 `extract_shining_point` 中 -5 分，可能导致即使素材非常适合"数据对比"或"捡漏"也被强行扭转到核心赛道。长期可能错过真实的爆款机会。

10. **验证器和爽点评分器都只测形式**——所有维度都是正则关键词匹配，不涉及语义理解。爽点评分器的 5 种结构能检测「信息密度」，但检测不了「信息是否准确」「论点是否成立」。目前正则的召回率和准确率未经大规模测试。

11. **爽点评分器无法检测「凭空编造」**——模板填充可能为了凑数字落差而编造数据（模板中无硬编码数字，但 `auto_fix` 追加段落可能引入虚构数据）。需要额外的事实核查机制。

12. **数字落差检测受单位限制**——只检测同单位数字对（万元 vs 万元），无法识别跨单位对比（如"单价降了 2 万，100 平的房子就省了 200 万"）。
