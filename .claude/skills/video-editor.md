# 视频自动剪辑 v3.0

融合两套工作流：
- **Bonnie 分步审核流**：转写→校对→计划→审核→渲染→反馈→迭代，确保准确度
- **知识型剪辑流（v3.0 新增）**：数据卡片/对比表格/金句全屏/系列统一/文化感节奏。小区深度分析、板块对比、数据解读类内容默认走此流

> Serena 代码动画流（Remotion）已移除——当前模型不支持视觉识别，截图复刻不可用。如需打字机/弹跳动画等效果，后期再考虑。

## v2 → v3 核心变化

| 维度 | v2 | v3 |
|------|-----|-----|
| 内容适配 | 泛类型，无区分 | **知识型/陪伴型自动区分**，两套剪辑参数 |
| BGM | 轻快现代纯音乐 25% | **知识型：舒缓钢琴/吉他 18-22%** |
| 字体 | 思源黑体 | **黑体(字幕)+宋体(标题/金句)** |
| 字幕 | 常规字幕 | **+数据卡片+对比表格+金句全屏** |
| 音效 | 2-4个/分钟 | **知识型：0-2个/分钟，宁缺毋滥** |
| 节奏 | 正常节奏 | **知识型：更慢、更多留白、金句后停留** |
| 系列化 | 无 | **统一标识/字体/BGM/布局** |

## 触发方式

用户说以下任意一句即触发：
- "剪这条" / "帮我剪辑" / "自动剪辑"
- "按脚本把这个素材剪出来"
- "生成剪辑指令" / "这条怎么剪" / "出分镜表"

## ⛔ 模型能力边界（必须先读）

**当前使用的模型不具备视觉识别能力**，因此：

| 能做到 | 做不到 |
|--------|--------|
| 语音转文字、对齐脚本 | 看视频画面判断"这是客厅还是主卧" |
| 按时间戳裁剪、拼接、变速 | 自动挑"采光最好的那个镜头" |
| 烧录字幕、数据卡片、文字叠加、BGM混音 | 分析画面内容加地名标签 |
| 生成 ASS 字幕实现强调/颜色/大小变化 | 看成品视频判断"这个转场好不好看" |

**核心原则**：我负责执行，你负责审美。你看了成片说改哪里，我改。我不自说自话评价效果。

## 工作流总览

```
内容类型判断 → 素材准备 → 转录校对 → 出剪辑计划 → 用户审核 → 渲染输出 → 看片反馈 → 迭代
```

**重要**：默认走分步审核流程。用户明确说"一步到位直接出片"才跳过中间审核。

## 第〇步：内容类型判断（v3.0 新增）

读取脚本 frontmatter 的 `strategy` 和 `pillar` 字段，自动判定剪辑风格：

| 判定条件 | 剪辑风格 | 参数预设 |
|----------|----------|----------|
| strategy=专业分析 且 pillar=Property | **知识型** | 舒缓BGM、宋体金句、数据卡片、少音效、慢节奏 |
| strategy=专业分析 且 pillar=Market | **知识型** | 同上，数据图表更多 |
| strategy=避坑指南 | 偏知识型 | 偏沉静BGM，保持数据强调 |
| strategy=陪伴决策 且 pillar=Story | 陪伴型 | 轻快BGM、黑体、正常音效、正常节奏 |

**不确定时**：默认按知识型处理（宁可偏沉静也不要偏浮躁）。

## 第二步：转录校对（Bonnie 流程核心）

### 2.1 转录

```bash
py -3 -m whisper <素材路径> --language zh --model base --output_dir <临时目录> --output_format txt
```

或读取已有脚本。

### 2.2 输出台本给用户校对

```
## 素材转录结果

| 时间段 | 文本 |
|--------|------|
| 0:00-0:03 | 苏州河拐进天山板块这一段 |
| 0:03-0:08 | 沿岸四个小区... |
| ... | ... |

⚠ 请校对以上文本：
- 有没有识别错误需要修正？
- 有没有需要删除的废话段落？
- **数据、地名、数字是否正确？**（知识型内容的数字错误是致命的）
- 确认无误后说"可以，继续下一步"
```

### 2.3 如果有脚本

```bash
py -3 video_editor.py <脚本路径> <素材路径> --align-only
```

## 第三步：出剪辑计划

### 3.1 询问用户需求（知识型默认值）

```
请确认以下剪辑需求（直接回复"都按默认"即可）：

1. 字幕样式？ [知识型默认：黑体字幕+宋体标题/金句]
2. 数据展示方式？ [知识型默认：关键数字卡片叠加+对比表格]
3. BGM？ [知识型默认：钢琴独奏，音量20%，闪避开启]
4. 音效？ [知识型默认：极少，至多2-3个极简强调音]
5. 成片节奏？ [知识型默认：稍慢，金句段留白]
```

### 3.2 输出剪辑计划表

```
## 剪辑计划

**内容类型**：知识型（小区深度分析）
**素材时长**：X 分 Y 秒
**预估成片**：约 X 分 Y 秒

### 分段安排

| 段 | 原始时间 | 成片时间 | 操作 | 字幕 | 画面叠加 |
|----|----------|----------|------|------|----------|
| 开场 | 0:00-0:05 | 0:00-0:04 | 保留 | 黑体白字 | 价格对比数据卡片 |
| 基本面 | 0:05-0:18 | 0:04-0:16 | 保留 | 黑体白字 | 小区数据卡片 |
| 优点二 | 0:28-0:50 | 0:25-0:47 | **此处放慢** | 黑体白字 | 澎湃截图+"84.5%"强调 |
| ... | ... | ... | ... | ... | ... |

### 字幕配置（知识型）
- 常规字幕：思源黑体 Regular，60px，白色+黑色2px描边
- 数据强调：思源黑体 Medium，75-80px，金色 #FFD700
- 章节标题：思源宋体 Bold，65px，居中
- 全屏金句：思源宋体 Bold，100px，居中，画面暗角

### BGM 配置（知识型）
- 风格：钢琴独奏
- 音量：人声段 18-20% / 纯画面段 25-28%
- 闪避：开启（人声时自动降至 18%）
- 淡入淡出：各 1-1.5 秒
- 金句段：BGM 提前 1 秒渐强至 22%，说完后维持 2 秒再淡出

### 音效（知识型）
- 至多 2 个极简强调音
- 不用 whoosh/pop/comedy 类音效
- 如果拿不准，不加

### 数据叠加计划
| 时间点 | 类型 | 内容 | 样式 |
|--------|------|------|------|
| 0:03 | 对比卡片 | 新天地10.8万 vs 仁恒9.7万 vs 河畔9.3万 | 半透明黑底+金色数字 |
| 0:12 | 数据四格 | 2011年 | 10.8万/㎡ | 650万起 | 1:1.2 | 半透明卡片 |
| 0:35 | 数字强调 | 84.5% | 金色大字弹出 |
| 结尾 | 全屏金句 | "房子会旧..." | 宋体Bold+画面暗角 |

⏸ 请审核以上计划。回复"确认执行"开始渲染，或指出要调整的地方。
```

## 第四步：渲染

```bash
# 知识型内容
py -3 video_editor.py <脚本路径> <素材路径> --output <输出路径> --font <黑体.ttf>

# 带 BGM
py -3 video_editor.py <脚本路径> <素材路径> --output <输出路径> --font <黑体.ttf> --bgm <BGM路径>
```

输出路径默认：`$env:USERPROFILE\Desktop\<脚本名>-成片.mp4`

完成后告知：成片路径、成片时长、"请在手机上观看，告诉我哪里需要调整"。

## 第五步：看片反馈 + 迭代

| 反馈 | 处理 |
|------|------|
| "节奏太慢/太快" | 调整段落时长或语速参数 |
| "数据卡片停留太长/太短" | 调整叠加持续时长 |
| "BGM 太响/太轻/风格不对" | 调整音量或换BGM |
| "字幕字体/大小/位置" | 调整 ASS 字幕参数重新烧录 |
| "金句不够突出" | 加大字号、加强暗角、延长停留 |
| "这个数字要强调" | 该段时间点加数据卡片或金色字幕 |

## 知识型 vs 陪伴型：参数速查

| 参数 | 知识型 | 陪伴型 |
|------|--------|--------|
| BGM 风格 | 钢琴/吉他独奏 | Lo-fi/轻快电子 |
| BGM 音量 | 18-22% | 25-30% |
| 字体搭配 | 黑体(字幕)+宋体(标题) | 黑体或圆体 |
| 金句字号 | 100-110px | 85-95px |
| 数据展示 | 卡片/对比表格/金色数字 | 偶尔强调即可 |
| 音效频率 | 0-2个/分钟 | 2-4个/分钟 |
| 金句停留 | 说完+1.5s | 说完+0.5s |
| 淡入淡出 | 1-1.5s | 0.5-1s |
| 系列标识 | 有（统一位置） | 无 |

## 质量保障

- Whisper base 对纯普通话效果好，方言/中英混杂会下降。转录完成后**必须**让用户校对
- 中文必须指定字体文件，否则可能乱码。知识型内容建议双字体
- 知识型内容的数字错误是一票否决的——校对时优先检查数字
- 渲染后检查文件大小 > 0、时长在预估范围 ±30%
