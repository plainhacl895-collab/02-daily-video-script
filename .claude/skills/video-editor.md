# 视频自动剪辑 v2.0

融合两套成熟工作流：
- **Bonnie 分步审核流**：转写→校对→计划→审核→渲染→反馈→迭代，确保准确度
- **Serena 代码动画流**：截图复刻、音效字幕、打字机B-roll等动画效果（需 Remotion）

## 触发方式

用户说以下任意一句即触发：
- "剪这条" / "帮我剪辑" / "自动剪辑"
- "按脚本把这个素材剪出来"
- "生成剪辑指令" / "这条怎么剪"
- "复刻这个视频的剪辑风格" / "帮我做一个这样的动画效果"

## ⛔ 模型能力边界（必须先读）

**当前使用的模型不具备视觉识别能力**，因此：

| 能做到 | 做不到 |
|--------|--------|
| 语音转文字、对齐脚本 | 看视频画面判断"这是客厅还是主卧" |
| 按时间戳裁剪、拼接、变速 | 自动挑"采光最好的那个镜头" |
| 烧录字幕、文字叠加、BGM混音 | 分析画面内容加地名标签 |
| 截图→读取图片→复刻布局/颜色/动画风格 | 录屏→分析动画逻辑→复刻 |
| 写 Remotion 代码生成动画效果 | 看成品视频判断"这个转场好不好看" |

**核心原则**：我负责执行，你负责审美。你看了成片说改哪里，我改。我不自说自话评价效果。

## 工作流总览

```
素材准备 → 转录校对 → 出剪辑计划 → 用户审核 → 渲染输出 → 用户看片反馈 → 迭代修改
```

**重要**：默认走分步审核流程。用户明确说"一步到位直接出片"才跳过中间审核。

## 第一步：素材准备

### 用户需要准备的文件

| 文件 | 必需？ | 说明 |
|------|--------|------|
| 口播素材 .mp4 | 必需 | 手机拍的就行，人声清晰最重要 |
| 中文字体 .ttf/.otf | 强烈建议 | 否则字幕中文可能乱码。推荐思源黑体，去 Google 搜"思源黑体 下载" |
| 背景音乐 .mp3 | 可选 | 需要的话提供，不提供则成片无BGM |
| 脚本 .md | 有就用 | 有 daily-script 最好，没有也能纯靠 Whisper 转录 |

### 字体问题说明

FFmpeg 烧录中文字幕需要系统里有中文字体。如果没有指定字体文件，会用系统默认字体，中文可能出现方块乱码。

**解决方案**：
1. 用户下载一个中文字体（.ttf 或 .otf），放到素材同目录
2. 告诉 Claude 字体文件名
3. FFmpeg 命令中指定 `fontfile=xxx.ttf`

## 第二步：转录校对（Bonnie 流程核心）

**这是最关键的步骤。不跳过。**

### 2.1 转录

```bash
py -3 -m whisper <素材路径> --language zh --model base --output_dir <临时目录> --output_format txt
```

或读取已有脚本（daily-script .md）。

### 2.2 输出台本给用户校对

将完整转写文本（带时间戳）打印给用户。格式：

```
## 素材转录结果

| 时间段 | 文本 |
|--------|------|
| 0:00-0:03 | 物业费九块五贵不贵 |
| 0:03-0:08 | 有个客户跟我说太贵了 |
| ... | ... |

⚠ 请校对以上文本：
- 有没有识别错误需要修正？（如"Clock耗" → "Claude Code"）
- 有没有需要删除的废话段落？
- 确认无误后说"可以，继续下一步"
```

### 2.3 用户校对

等待用户反馈。常见修正：
- 同音字错误（Whisper 对中文同音字敏感）
- 英文/专有名词识别错误
- 标记要删除的段落

### 2.4 如果有脚本

如果用户提供了 daily-script .md，跑对齐：

```bash
py -3 video_editor.py <脚本路径> <素材路径> --align-only
```

输出每段匹配质量。匹配度 < 0.5 的标红提醒用户关注。

## 第三步：出剪辑计划

用户确认转录无误后，根据用户需求输出剪辑计划表。

### 3.1 询问用户需求

如果用户没有明确说明，主动询问：

> 请确认以下剪辑需求（直接回复"都按默认"即可）：
> 1. 剪掉长时间停顿和口头禅（嗯/啊/就是说）？ [默认：是]
> 2. 字幕样式偏好？（白色+黑边 / 其他颜色）[默认：白色黑边]
> 3. 是否需要动画效果？（打字机/强调文字/对比列表）[默认：关键数字处加强调]
> 4. BGM 如何处理？（提供文件 / 不需要 / 让我推荐风格）[默认：不需要]
> 5. 成片时长偏好？（尽量短 / 正常节奏 / 保留完整）[默认：正常节奏]

### 3.2 输出剪辑计划表

```
## 剪辑计划

**素材时长**：X 分 Y 秒
**预估成片**：约 X 分 Y 秒（删减 Z 秒废话/停顿）

### 分段剪辑安排

| 段 | 原始时间 | 成片时间 | 操作 | 字幕 | 动画 |
|----|----------|----------|------|------|------|
| 1 | 0:00-0:08 | 0:00-0:06 | 保留，剪掉开头2秒空白 | 白字黑边 | 无 |
| 2 | 0:08-0:15 | 0:06-0:12 | 保留 | 白字黑边 | 无 |
| 3 | 0:15-0:18 | — | 删除（口头禅"就是说"） | — | — |
| ... | ... | ... | ... | ... | ... |

### 字幕配置
- 字体：<字体文件名>
- 大小：55（常规）/ 80（强调）/ 90（金句）
- 颜色：白色 + 黑色描边
- 位置：底部（常规）/ 居中（金句）

### BGM 配置
- 文件：<bgm文件名或"无">
- 音量：30%（人声段）/ 40%（纯画面段）
- 闪避：开启（人声时自动降至 30%）
- 淡入淡出：1秒

### 动画安排（如适用）
| 时间点 | 类型 | 内容 | 触发条件 |
|--------|------|------|----------|
| 0:12 | 数字强调 | "五十万 vs 八万四" | 脚本 [字幕叠加] 标注 |
| 0:22 | 全屏金句 | "物业费不是消费..." | 脚本金句段落 |

⏸ 请审核以上计划。回复"确认执行"开始渲染，或指出要调整的地方。
```

## 第四步：渲染

用户确认计划后执行渲染。

### 4.1 基础渲染（纯 FFmpeg）

用 video_editor.py 渲染：

```bash
py -3 video_editor.py <脚本路径> <素材路径> --output <输出路径>
```

如有 BGM：
```bash
py -3 video_editor.py <脚本路径> <素材路径> --output <输出路径> --bgm <BGM路径>
```

输出路径默认：`$env:USERPROFILE\Desktop\<脚本名>-成片.mp4`

### 4.2 动画渲染（Remotion，可选）

如果用户需要复杂动画效果（打字机、音效字幕、截图复刻），需要 Remotion。

**检查 Remotion 是否已安装**：
```bash
npx remotion --version
```

如果未安装，告知用户：
> 这个动画效果需要 Remotion。安装只需一条命令（约2分钟）：
> ```bash
> npx skills add remotion-dev/skills
> ```
> 安装后告诉我，我继续。

已安装的话，按用户需求写 Remotion React 代码 → `npx remotion render` 渲染。

### 4.3 渲染完成后

告知用户：
- 成片路径
- 成片时长
- 每段处理结果摘要
- "请在手机上用相册App观看，告诉我哪里需要调整"

## 第五步：看片反馈 + 迭代

用户看完成片后会给出反馈。常见反馈类型：

| 反馈 | 处理 |
|------|------|
| "开头太慢/太快" | 调整第一段的 trim 起点或语速 |
| "字幕太小/太大/位置不对" | 调整 ASS 字幕参数，重新烧录 |
| "这段不要" | 从 timeline 中删除该段，重新 concat |
| "这里加个强调" | 该段时间点加 scale 110% 或 emphasis 字幕 |
| "BGM 太响/太轻" | 调整 volume 参数 |
| "这段动画效果不满意" | 改 Remotion 代码参数，重新渲染 |

每次迭代后重新输出，直到用户满意。

## 模式 2：生成剪辑指令（还没拍素材）

用户只有脚本还没拍摄时，生成精确的逐秒分镜表。

**执行**：
1. 解析脚本所有 `[画面]`、`[字幕叠加]`、`[语速]` 标注
2. 按脚本字数 + 语速标注估算每段时长
3. 输出剪辑指令单：

```
## 剪辑指令单

| 时间段 | 时长 | 画面 | 字幕叠加 | 语速 | 拍摄要点 |
|--------|------|------|----------|------|----------|
| 0:00-0:04 | 4s | 正视镜头，钩子 | 无 | 正常 | 一句话干净利落 |
| 0:04-0:20 | 16s | 自然讲述 | "差额8.4万/10年"叠层 | 慢 | 每个数字停一秒 |
| 0:20-0:25 | 5s | 正视镜头，认真 | 全屏"物业费不是消费..." | 慢 | 说完停顿留白 |
| 0:25-0:30 | 5s | 微笑收尾 | "关注我"引导 | 正常 | 自然收尾 |
```

## 模式 3：动画效果生成（Remotion）

用户提供截图或描述，生成带动画效果的视频片段。适用于混剪视频、数据对比卡片、品牌片头。

### 3.1 截图复刻

用户提供一张参考截图 → Claude 读取画面 → 分析布局/颜色/文字/动画 → 写 Remotion 代码复刻。

**仅当使用多模态模型（Claude Opus/Sonnet）时可用。** 当前 deepseek 模型不支持读取截图复刻动画。

### 3.2 描述生成

用户用自然语言描述效果 → Claude 写 Remotion 代码 → 渲染。

例如：
- "做一个打字机效果的板书，文字逐字跳出，配机械键盘声"
- "做一个数字对比卡片：'五十万 vs 八万四'，左边的数字放大变金色"
- "做一个视频章节进度条，随播放进度填充"

## 质量保障

### 转录质量
- Whisper base 对纯普通话效果好，方言/中英混杂会下降
- 环境噪音大（风噪、马路）会显著降低准确率
- 转录完成后**必须**让用户校对

### 字幕质量
- 中文必须指定字体文件，否则可能乱码
- 字幕断句按标点 + 字数估算，长句可能不自然
- ASS 字幕支持 `\move` 位移、`\fade` 淡入淡出、`\t` 动态变化

### 渲染质量
- 渲染后检查文件大小 > 0、时长在预估范围 ±30%
- 时长偏差 > 50% 说明段落匹配可能出问题

### 已知限制
- 不能自动判断画面内容（模型无视觉能力）
- 不能做粒子特效、3D 转场（需要 AE/剪映）
- 不能做调色、美颜
- FFmpeg 做不了逐字弹跳动画（需要 Remotion）
