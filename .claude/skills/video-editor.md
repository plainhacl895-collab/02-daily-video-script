# 视频自动剪辑

从脚本标注到成片输出，替代手工剪映操作。

## 触发方式

用户说以下任意一句即触发：
- "剪这条" / "帮我剪辑" / "自动剪辑"
- "按脚本把这个素材剪出来"
- "生成剪辑指令" / "这条怎么剪"
- "分析这个视频的剪辑风格"

## 四种模式

### 模式 1：全自动剪辑（有脚本 + 有素材）

```
脚本 (.md) + 口播素材 (.mp4) → 自动成片
```

**执行流程**：

1. **确认输入**：脚本路径 + 素材路径。如果用户只给了其中一个，主动询问另一个。
2. **运行对齐**：
   ```bash
   python video_editor.py <脚本路径> <素材路径> --align-only
   ```
   输出 JSON，包含每段在素材中的精确时间戳。
3. **质量审查**：检查对齐结果——
   - 匹配度（match_ratio）≥ 0.5 → 通过
   - 匹配度 < 0.5 → 告知用户哪段匹配差，建议手动指定时间
   - 如果某段 duration = 0 → 说明脚本文字和口播差距太大（可能拍摄时即兴改词了），需要人工确认
4. **渲染**：
   ```bash
   python video_editor.py <脚本路径> <素材路径> --output <输出路径>
   ```
   如果有 BGM 文件，加 `--bgm <BGM路径>`。
   输出路径默认桌面：`$env:USERPROFILE\Desktop\<脚本名>-成片.mp4`
5. **结果汇报**：告知用户成片位置、总时长、每段处理情况。

### 模式 2：对齐检查（不确定素材质量时）

只做对齐，不渲染。让用户先看时间线是否合理，再决定是否渲染。

```bash
python video_editor.py <脚本路径> <素材路径> --align-only
```

输出给用户：
- 每段在素材中的起止时间
- 预估成片时长
- 每段匹配质量（高/中/低）
- 如果匹配质量低，给出具体建议（哪段需要重拍、哪段可以手动调）

用户确认后，用模式 1 渲染。

### 模式 3：生成剪辑指令（还没拍素材）

用户只有脚本，还没拍摄。生成一份精确的剪辑指令单，用户拍完后可以照着剪（手动或用模式 1 自动）。

**执行**：
1. 读取脚本文件，解析所有 `[画面]`、`[字幕叠加]`、`[语速]` 标注
2. 基于脚本 word_count_target 估算每段时长
3. 输出一份**逐秒分镜剪辑表**：

```
## 剪辑指令单

| 时间段 | 时长 | 画面 | 字幕叠加 | 语速 | FFmpeg 操作 |
|--------|------|------|----------|------|-------------|
| 0:00-0:04 | 4s | 正视镜头，钩子 | 无 | 正常 | trim, copy |
| 0:04-0:18 | 14s | 自然讲述 | 关键数字叠层 | 慢 | trim, setpts=1.15*PTS, drawtext |
| 0:18-0:22 | 4s | 正视镜头，认真 | 全屏金句 | 慢 | trim, setpts=1.15*PTS, scale 110%, drawtext 大字 |
| 0:22-0:28 | 6s | 微笑收尾 | 关注引导 | 正常 | trim, copy, drawtext |
```

### 模式 4：参考视频风格拆解

用户给一个参考视频链接 + 自己的脚本，生成"同款剪辑方案"。

**执行**：
1. 用 `douyin_extractor.py <链接> --asr` 提取参考视频口播文案
2. 逐秒分析参考视频的剪辑节奏：
   - 镜头切换频率（几秒一切）
   - 字幕出现时机（开头/中间/结尾？持续多久？）
   - 强调效果位置和类型（放大/变色/振动？）
   - BGM 节奏与画面的配合
   - 转场类型和使用频率
3. 将用户的脚本按参考视频的节奏重新分镜
4. 输出一份"同款剪辑分镜表"，格式同模式 3

## 质量保障

### 对齐失败处理

如果 Whisper 对齐质量差（某段 match_ratio < 0.3），不要强行渲染。原因通常是：
- 脚本写了但拍摄时改了词
- 拍摄时有大段即兴发挥
- 环境噪音导致转写错误

处理方式：
1. 告诉用户具体哪段对齐失败
2. 让用户手动指定该段在素材中的起止时间（"你帮我看看，开场那段在视频里大概第几秒到第几秒"）
3. 收到用户的时间后，手动修正 timeline 再渲染

### 渲染质量检查

渲染完成后，检查：
- 视频文件大小 > 0（没渲染失败）
- 时长在脚本预估范围内（±20% 可以接受）
- 如果时长偏差 > 50%，检查是否有段落匹配错误

### 已知限制

- Whisper base 模型对上海话/方言混合口播识别较差，纯普通话效果好
- 画面放大（emphasis）目前是简单的 scale + crop，效果比剪映的"关键帧缩放"粗糙
- 字幕断句按标点 + 字数估算，长句可能断得不够自然
- 不支持调色、美颜、转场特效——这些仍需剪映手动做

## 安装依赖

首次使用前确认以下工具可用：

```bash
# 检查 FFmpeg
ffmpeg -version

# 检查 Whisper（Python）
python -c "import whisper; print(whisper.__version__)"
```

如果 Whisper 未安装：
```bash
python -m pip install openai-whisper
```

## 与人设体系的关系

这个技能只做**技术性剪辑**（切、速、字幕、对准），**不参与内容决策**。所有创意判断（钩子好不好、金句放哪里、语气对不对）由脚本生成流程负责，不在剪辑阶段改动。
