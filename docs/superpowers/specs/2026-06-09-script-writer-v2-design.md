# Script Writer Skill v2 — 情绪维度 + 学习路径

日期：2026-06-09 | 状态：待审批

## 背景

v1.0 的 9 步工作流产出脚本逻辑严谨、数据扎实，但在情绪感染力和讨论激发力上为零。根本原因：

1. **中心思想定太窄**：一个数据点当核心论点，60秒讲完，后面全是硬撑
2. **没有情绪维度**：流程里没有任何环节要求考虑"观众看完会有什么感受"
3. **幽默是算出来的**：在数据堆里硬塞"轻松的点"，不自然
4. **我的能力边界**：擅长逻辑和数据，不擅长情绪和幽默

## 改造目标

在不推翻现有 9 步流程的前提下，把"情绪驱动"能力嵌入到工作流中，同时建立学习机制让能力随使用积累。

## 四大改动

### 1. 情绪基调（Step 5→6 之间新增）

叙事结构设计完成后，必须标注「情绪基调」——这条脚本想让观众看完产生**哪一种感受**。

可选类型（不限于此）：
- "学到了一个方法"（工具型）
- "被说中了"（共鸣型）
- "原来是这样"（认知刷新型）
- "替自己捏把汗"（风险揭示型）

情绪基调由我提议，**你必须点头**才能进入写脚本环节。

### 2. 节奏断点计划（Step 5→6 之间新增）

每约 120 字标记一个节奏变化点，我只标注**位置和功能**（如 `[节奏断点：过渡到下一个数据]`），具体内容由你即兴发挥或从钩子库选。

设计要求：
- 不从逻辑里硬造，从信息里自然引出
- 可以是提问、观察、对比、停顿——根据上下文决定
- 我在标注时只写功能描述，不写具体台词

### 3. 话题钩子检查（Step 6 末尾新增）

结尾提问必须是**经历型**，禁止空泛型。

- 好："你有没有看房的时候遇到过XX？"
- 坏："你怎么看？"

原理：经历比观点门槛低，观众更容易参与。

### 4. 学习路径（触发条件 + 新增输出）

新增触发词：`看看这条` / `分析一下这个视频`

你扔视频给我 → 我用现有工具提取内容 → 输出两份笔记：

| 输出 | 写入位置 | 内容 |
|------|---------|------|
| 竞品技法分析 | `competitor-learning-log.md`（已有） | 选题角度、文案结构、拍摄手法 |
| 情绪钩子提取 | `emotional-hooks.md`（新建） | 可复用的节奏模式、调侃方式、表达技巧 |

情绪钩子入库格式（每条 ≤4 行）：

```markdown
### 钩子名称

- **来源**：<视频链接>
- **模式**：<可复用的表达方式，一句话>
- **适用场景**：<什么时候能用>
```

来源不限领域——知识区、脱口秀、纪录片、访谈均可。拆的是表达技巧，不是专业知识。
