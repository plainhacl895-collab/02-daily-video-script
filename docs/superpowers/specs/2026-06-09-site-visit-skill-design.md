# 踩盘技能设计 — site-visit.md

日期：2026-06-09 | 状态：待实现

## 背景

线上信息分两种：客观数据（大家都能查到，念出来没劲）和同行分析（相互抄，缺少深度）。实地踩盘看到的细节——保安状态、下午采光、公区维护、周边真实界面——是线上永远查不到的独家素材。

踩盘一趟成本高，不能逛了就走。需要带着问题出门，回来有东西沉淀。

## 技能文件

`.claude/skills/site-visit.md`，独立技能。

## 触发条件

用户说"明天去踩XX小区"/"准备去看XX板块"/"要去踩盘"/"实地看一下XX"。

## 三步流程

### Step 1 — 线上预研

在出发前，我先做一轮线上搜索和信息整理：

1. 读已有分析文件（小区深度分析 + 板块分析）
2. 搜索目标小区/板块的线上信息（贝壳、链家、小红书、安居客）
3. 跑 `python engine.py listings <小区名>` 拿实时挂牌数据
4. 输出 **「已知 vs 待验证」表**：

| 已知（线上可查） | 存疑/缺失（需要你现场验证） |
|----------------|--------------------------|
| 绿化率68.5% | 低楼层实际采光如何？20年大树遮到什么程度？ |
| 物业费缴纳率99.8% | 保安登记访客用了多久？喷泉水面上有落叶吗？ |
| 挂牌均价9.7万 | 在售房源的真实状态和照片差距 |

5. 提出一个**初步中心思想方向**（可以调整），作为设计清单的锚点

### Step 2 — 定制观察清单

基于「待验证」列 + 初步中心思想 + 踩盘行走路线，生成定制清单。

**路线顺序**（不是按维度）：

1. 小区外围/周边界面 — 从地铁站出口开始
2. 小区大门/安保 — 进门前就开始了
3. 小区内部环境 — 边走边看
4. 楼栋公区 — 大堂、电梯、楼道
5. 房源内部 — 终于进屋了
6. 周边走一圈 — 从房子出来散步

**每条问题的要求**：必须是具体的、可观察的、跟中心思想相关的。不是"看看环境"而是"从威宁路3号口出来走这段路，人行道宽度和沿路店铺类型如何？有没有不舒服的地方？"

**上限15条**，1-2小时踩盘刚好。

### Step 3 — 踩盘后录入

两种录入方式：

| 方式 | 做法 | 适合 |
|------|------|------|
| 语音 | 踩盘时或结束后对手机录音，想到什么说什么 | 记忆最新鲜时 |
| 口述 | 回来说给我，我问你答 | 想系统整理时 |

提炼内容写入：

| 位置 | 写什么 | 标记 |
|------|--------|------|
| 对应小区/板块分析文件 | 观察到的具体细节、数据、与线上差异 | `[实地验证] [日期]` |
| 可选的踩盘行动记录 | 一次行动的简要日志 | 不影响现有结构 |

**优先级规则**：以后写同小区脚本，`[实地验证]` 标签的数据优先于线上数据。

## 核心原则

- **清单为内容服务**：每个问题都在为脚本的中心思想搜集素材。同一个小区，不同中心思想，清单完全不同
- **不问感受，问事实**：不问"环境好不好"，问"下午3点阳光到客厅什么位置"
- **成本意识**：一趟出去成本高，必须带回来线上没有的东西
